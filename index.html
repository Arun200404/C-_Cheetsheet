Boxing, ref / out & Nullable

Most Used
// Nullable value type
int? age = null;
if(age.HasValue) Console.WriteLine(age.Value);
int safe = age ?? 0; // null-coalescing
age ??= 18;           // assign if null

// out: method MUST assign (very common)
void Get(out int x) { x = 20; }
Get(out n);
Other / Less Common
// ref: init before pass, bidirectional
void Add(ref int x) { x += 10; }
Add(ref n);  // n is modified

// Boxing: value → object (heap)
int a = 10;
object obj = a;   // implicit box
int b = (int)obj;  // explicit unbox
Avoid boxing in loops — heap alloc + GC cost. Type must match exactly or InvalidCastException.

Inheritance, Polymorphism & Interfaces

Most Used
// Single inheritance
public class Dog : Animal { }

// virtual + override (runtime polymorphism)
public virtual void Speak() { }      // base
public override void Speak() { }     // derived

// Call base class method
public override void Init() {
  base.Init(); // extend, not replace
}

// Interface
public interface IFly { void Fly(); }

// Multiple interfaces
class Eagle : Bird, IFly, IHunter { }

// Enum
enum Status { Pending=1, Done=5, Failed=10 }
Status s = Status.Done;
int v = (int)s; // 5
Other / Advanced
Abstract Class vs Interface
AbstractInterface
Fields / State✅❌
Constructors✅❌
Multiple inherit❌✅
Can instantiate❌❌
// Abstract class
public abstract class Shape {
  public abstract double Area();  // must implement
  public void ShowName() { }      // concrete
}

// Method hiding (NOT polymorphic!)
public new void Show() { }

// Explicit interface implementation
void IPrinter.Print() { } // must cast to use

// Sealed class — no further inheritance
sealed class Final { }
Override = runtime dispatch. new = compile-time hiding. Always prefer override.

► Threading, Async & NUnit

Threading, Parallel & Async

Async / Await (Most Used)
// void → Task, T → Task<T>
async Task DownloadAsync() {
  await Task.Delay(2000);
}
async Task<int> GetValueAsync() {
  await Task.Delay(500);
  return 42;
}
int v = await GetValueAsync();

// Parallel async (WhenAll)
var tasks = items.Select(i => ProcessAsync(i));
await Task.WhenAll(tasks);

// Task with result
Task<int> t = Task.Run(() => 42);
int val = await t;
No thread blocking. Best for file/network/DB I/O. Always await or exception is swallowed.
Parallel (CPU-bound)
using System.Threading.Tasks;

Parallel.For(1, 6, i => {
  Console.WriteLine(i);
});

Parallel.ForEach(list, item => {
  Console.WriteLine(item);
});

// Task (explicit)
Task task = Task.Run(() => {
  Console.WriteLine("Background");
});
await task;

// First to complete
await Task.WhenAny(t1, t2);
Parallel auto multi-core. Order not guaranteed. Use for CPU-intensive loops.
Thread (manual — less common)
using System.Threading;
Thread t = new Thread(DoWork);
t.Start();
Thread.Sleep(1000);
t.Join();   // wait to finish
t.IsAlive;  // bool
t.Priority; // ThreadPriority

// Lambda shortcut (closure)
Thread t3 = new(() => Process(data));
t3.Start();

// Parameterized thread
Thread t2 = new(o => {
  var order = (Order)o!;
});
t2.Start(orderObj);
Pass data via lambda closure — no ParameterizedThreadStart needed.

NUnit Testing

Most Used — Structure
[TestFixture]
public class CalcTests {
  private Calculator calc;

  [SetUp]
  public void Init() => calc = new();

  [TearDown]
  public void Cleanup() { }

  [Test]
  public void Add_Returns5() {
    Assert.AreEqual(5, calc.Add(2,3));
  }

  [TestCase(2, 3, 5)]
  [TestCase(0, 0, 0)]
  public void Add_Param(int a, int b, int exp)
    => Assert.AreEqual(exp, calc.Add(a,b));
}
